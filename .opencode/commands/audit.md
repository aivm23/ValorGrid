# /audit - Auditoría de ValorGrid

Uso: `/audit <scope>` donde scope es uno de:

- `release` - Antes de tag/release
- `ai` - Drift en comandos, skills, AGENTS y planes
- `docs` - Sincronización docs vs código
- `imports` - Foco en importación de datos
- `privacy` - Publicación segura
- `architecture` - Layering del monorepo
- `deps` - Dependencias
- `docker` - Docker/CasaOS

Sin argumento: diagnóstico superficial de todo lo anterior.
